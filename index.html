Stage 03
Visited
Appointment completed. Pull finalized during the showroom visit.

Stage 04
Out
Items have left the showroom. Return date set. Calendar event created and Event ID stored.
Stage 05 · Automated

Stage 06 · Automated
Overdue

Items return
Close Track
Stage 07
Returned
Items received back. Condition checked. Inventory status reset to available.
Stage 08
Closed
Coverage logged — event name, platform, credit given. Client reliability score updated.

Why $25/mo
Running AI agents requires a persistent, always-on process — the Render Standard tier ($25/mo) prevents the instance from sleeping between requests, which is essential for a Slack bot that needs to respond immediately. Cloud Starter covers the same execution volume and is fully managed, making it equally valid at the same price point.

07

RAG
AI Agent

A natural language assistant that sits on top of the CRM and inventory — so the team can ask questions in plain English instead of digging through spreadsheets, messages, or Airtable views.

01 — File Naming Convention

Every inventory photo in Google Drive follows a fixed structure. Each token is a parseable field — the agent reads the filename directly to understand what the item is without needing a separate database lookup.

Item ID001 · 042 · 007 — unique per item, links to CRM

03 — Example Queries
What the team can ask the agent — answered instantly from live CRM and inventory data.
"What do we currently have out?"
Returns all active Pull records with status Out or Due Soon — client name, items, and return dates.
"Who has Versace pieces right now?"
Filters inventory by designer, cross-references active pulls to return the client holding each item.
"Who did we work with at BET last year?"
Searches closed pull records filtered by coverage event field containing "BET" and the relevant year.
"Make a list of stylists with overdue returns"
Pulls all records in Overdue stage — client name, items out, days past due, last contact.
"What jackets do we have available in FW 2024?"
Filters inventory by item type and season, returns only items with status Available.
"Which clients returned on time last season?"
Queries closed pulls filtered by return date vs due date — surfaces reliability scores by client.
